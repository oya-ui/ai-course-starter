# AIコース — Step2（ストーリー・スコープ整理）

## 目的

`mvp/STEP1.md` の仮説を入力に、**誰が何の画面で何をするか** を1ドキュメントにまとめ、
Step3（UIモック）に渡せる状態にする。

---

## 入力の確認

最初に `mvp/STEP1.md` を読む（ない場合はユーザーに内容を貼ってもらう）。
読めたら Step1 の「主導線・削り後の機能・G3」を1〜2行で要約してから進む。

---

## 進め方（必須の順番）

1. **主導線の確認**: Step1 で決めた「A → B → C」を声に出して確認
2. **ストーリー**: トリガー（きっかけ）→ 操作ステップ → ゴール（成功状態）
3. **画面一覧**: S1 / S2 / S3 … の番号を振り、各画面で「何が確認できるか」を書く
4. **除外の再確認**: Step1 で削ったものがストーリーに紛れ込んでいないか確認
5. **スコープ段の確認**: A〜D から「まずどこまでやるか」を決める（下記）
6. **MVP チェックリストの確認**: 4項目すべて「Yes」になるか確認（下記）
7. **Step3 への引き渡しメモ**: 画面 ID と、各画面の要点を一行ずつ

---

## スコープ段 A〜D（インライン定義）

| 段 | 内容 | 目安 |
|----|------|------|
| A | コア主導線1本だけ。最小限のデータ保存 | まずここを目指す |
| B | A ＋ UX 微調整（バリデーション・エラー表示など） | A が動いてから |
| C | B ＋ 補助機能（検索・フィルタ・一覧改善など） | 余裕があれば |
| D | C ＋ 運用考慮（メール通知・管理画面・分析等） | 今回はスコープ外 |

→ **段 A を必達**として計画を立てる。「まず A、時間があれば B」と明確にする。

---

## MVP チェックリスト（Step2 完了基準）

| # | 確認 | Yes/No |
|---|------|--------|
| 1 | 主導線（A→B→C）が画面で追えるか | |
| 2 | データ保存（DB）またはログインがあるか | |
| 3 | 除外リストがあり、スコープが絞られているか | |
| 4 | Step3 に渡せる画面 ID 一覧があるか | |

4つ全部 Yes なら Step2 完了。

---

## 出力フォーマット（mvp/STEP2.md）

インタビュー完了後、**デフォルトで `mvp/STEP2.md` に保存する**。

```markdown
# MVP Step2 ストーリー・スコープ

## 1. 主役
- ユーザー: （誰）

## 2. ストーリー
トリガー: （きっかけ）
ステップ:
  1. （操作）
  2. （操作）
  3. ……
ゴール: （成功状態）

## 3. 画面一覧
| ID | 画面名 | この画面で確認できること |
|----|--------|------------------------|
| S1 | （名前） | （確認できること） |
| S2 | （名前） | （確認できること） |

## 4. 除外リスト
- （機能）: （除外理由）
- ……

## 5. スコープ段
- 必達: 段 A
- 余裕があれば: 段 B
- 今回スコープ外: 段 C / D

## 6. MVP チェックリスト
- [ ] 主導線が画面で追える
- [ ] DB またはログインがある
- [ ] 除外リストあり
- [ ] 画面 ID 一覧あり

## 7. Step3 への引き渡し
| 画面 ID | Step3 で作るモックのポイント |
|---------|---------------------------|
| S1 | （ポイント） |
| S2 | （ポイント） |

次のステップ → `/course-mvp-step3-ui-mock` を呼ぶ
```

---

## Step2 ⇄ Step3 の行き来

モック（Step3）を見てからストーリーを修正したくなることがある。
その場合は `mvp/STEP2.md` を更新し、Step3 に戻って構わない。

---

## やらないこと

- Step1 の仮説なしに新テーマをゼロから始めない
- HTML モックの作成はここでしない（それは Step3）
- Next.js の実装はここでしない（それは Step4）
