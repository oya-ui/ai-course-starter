# AIコース — Step3（UI モック作成 ＋ 改善ループ）

## 目的

`mvp/STEP2.md` の画面 ID とストーリーをもとに **静的 HTML 1ファイル**（`mvp/STEP3_UI_MOCK.html`）を作り、
**フィードバック → 修正 → 再確認** を繰り返してユーザーが「これでいい」と言うまで磨く。

---

## 入力の確認

最初に `mvp/STEP2.md` を読む（ない場合はユーザーに画面 ID と概要を教えてもらう）。
画面一覧（S1/S2…）と「各画面で確認できること」を把握してからモックを作る。

---

## 全体の流れ（ループ最大 5 往復）

```
[1] 初回モック作成（mvp/STEP3_UI_MOCK.html）
      ↓
[2] 改善インタビュー（下記の B1〜B5 を短く聞く）
      ↓
[3] ユーザーの指示を同一ファイルに反映
      ↓
[4] [2] に戻る
      ↓
[5往復終了時点] → 「あと1点だけ修正して完了にしましょうか？」と確認
      ↓
[了承] → Step3 完了を宣言
```

「これでいい」「OK」「いいね」「もうない」など了承の言葉が出たら即終了。

---

## HTML の作り方（1ファイル完結）

複数画面を1ファイルで表現するには、**タブ切り替え or セクション分け**を使う。

```html
<!-- 画面切り替えの例（インラインJSで実装） -->
<nav>
  <button onclick="show('s1')">S1: ログイン</button>
  <button onclick="show('s2')">S2: 一覧</button>
</nav>
<section id="s1"><!-- ログイン画面の中身 --></section>
<section id="s2" style="display:none"><!-- 一覧画面の中身 --></section>
<script>
function show(id) {
  document.querySelectorAll('section').forEach(s => s.style.display = 'none');
  document.getElementById(id).style.display = '';
}
</script>
```

- スタイルは `<style>` タグにまとめて書く（外部 CSS 不要）
- フォントは `font-family: system-ui, sans-serif` だけで十分
- モバイル対応は `<meta name="viewport" ...>` と `max-width: 600px; margin: auto;` で対応

## 改善インタビュー（フェーズ B）

毎ループ、以下を**短く**聞く（全部聞かなくて良い、気になるものだけ）。

| # | 確認ポイント |
|---|-------------|
| B1 | 主導線（最初の操作→ゴール）は Step2 のストーリーと同じ流れに見えるか |
| B2 | 各画面で確認したい情報は足りているか |
| B3 | 全体のトーン（シンプル・業務っぽい・カジュアル）は意図に合うか |
| B4 | 口頭で説明しにくい部分はあるか |
| B5 | 文字サイズ・ボタン位置など、すぐ直せる見た目の希望は |

→ 答えをもらったら**すぐ同一ファイルを更新**（ユーザーが「保存して」と言わなくても保存する）。

---

## ファイル保存の方針

- **初回作成・修正ともにデフォルトで `mvp/STEP3_UI_MOCK.html` に上書き保存**する。
  （「保存しなくていい」と言われたらチャット内コードブロックにとどめる）
- Step2 のスコープを大きく変える修正を求められたら「それは Step2 の変更になります」と伝える。

## やらないこと

- Step2 の画面 ID なしにモックだけ作らない
- 初回作成で終わらない（了承まで必ずループを続ける）
- Next.js のコード・API・DB 設定はここでしない（それは Step4）
- 5往復を超えてループを続けない（5往復目以降は「あと1点」モードで締める）
